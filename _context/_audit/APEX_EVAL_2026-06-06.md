# NoBC OS — Engineering Evaluation

_Apex (eng lead) + Spine / Flux / Warden / Proof · 2026-06-06_

## What this is

Multi-tenant member-ops platform for No Bad Company. Handles applications,
approvals, event RSVPs, ticketing, check-in, DAM/media, sponsor intelligence,
and an operator AI agent. Tenant Zero = NoBC.

- **Stack:** Next.js 15 (App Router, Turbopack), TypeScript, Tailwind v3, Prisma 7,
  Postgres/Neon, Clerk (Organizations), Stripe (auth/capture), Resend, Anthropic
  Claude, AWS S3.
- **Scale:** 739 tracked files, 544 TS/TSX, **138 API routes**, **49 Prisma models**,
  21 migrations.
- **Topology:** Postgres is SHARED by three writers — nobc-os, "Producer" (Replit
  operator ops), and a Runtype AI agent layer. One DB role, no schema boundary, no RLS.
- **Practical state:** single-tenant-in-practice (NoBC), built multi-tenant.

## Prioritized gaps

### P0 — fix this week

1. **`QAMission` migration unapplied** — `20260520130000_qa_missions` exists in
   schema + migrations but NOT in the DB. Any code path hitting `QAMission` throws
   `relation does not exist`. `npx prisma migrate deploy` before the QA panel ships.
   _(Flux RED #5)_
2. **`CHECKIN_SECRET` shipped to the browser** — `NEXT_PUBLIC_CHECKIN_SECRET`
   bakes the check-in bearer token into the JS bundle (`CheckInClient.tsx:9`,
   `WalkinModal.tsx:6`, `EventCheckinTab.tsx:24`). Anyone can pull it from devtools
   and enumerate any event's full guest list + register arbitrary walk-ins as
   APPROVED. Move to a short-lived, event-scoped signed token minted server-side
   from the Clerk session. _(Warden HIGH)_
3. **`rsvp/plus-one` approval bypass** — `app/api/rsvp/plus-one/route.ts:63` creates
   the guest as `status:'APPROVED', approved:true`, so any approved member can
   promote arbitrary emails to full APPROVED member status, bypassing review. Also
   no Zod, unbounded `guestName` interpolated raw into Resend HTML (injection). Use a
   `GUEST` status + Zod + length caps. _(Warden MEDIUM, real)_
4. **No CI + 0% API test coverage + ungated prod deploy** — 6 unit files cover pure
   utilities; **none** of the 138 routes (auth, Stripe, apply, check-in, webhooks)
   has a test. No `.github/workflows`, no Husky. A broken auth/payment change ships
   silently. Add a CI gate (`tsc --noEmit` + `vitest` + lint) on push, then write
   integration tests for the money paths first. _(Proof RED #1, #2)_

### P1 — this milestone

5. **Apple wallet unsigned token** — `wallet/apple/[rsvpId]/route.ts:35` token is
   `btoa("rsvpId:workspaceId")`. Both are observable CUIDs → forge any member's pass.
   HMAC-sign + verify. Flagged in prior audit, still open. _(Warden HIGH)_
6. **Shared-DB has no ownership boundary** — Producer writes `Event.producerEventId`,
   Runtype writes `AgentConversation`/`AgentTurn`, all via one Postgres role, no RLS,
   no foreign schema. One bad migration from any writer corrupts shared tables.
   Define migration ownership + give Producer/Runtype scoped roles or an API instead
   of direct writes. _(Spine #9, Flux #9)_
7. **Missing FK indexes on hot paths** — `RSVP.memberId` (member RSVP history =
   table scan), `Application.memberId`, `TicketHold.eventId`, `SmsConversation.eventId`.
   Latent now, real at 100k RSVPs. _(Flux #8)_
8. **Serverless statelessness** — module-level `Map` caches and the AI rate limiter
   (`lib/agent/lib/rate-limit.ts`) don't survive/​share across Vercel instances → the
   AI endpoints are effectively unthrottled at scale. No timeout/`AbortSignal` on
   outbound Anthropic + Producer webhook calls → stalled calls hold function slots.
   _(Spine #6, #8)_
9. **DAM tag endpoint fails open** — `media/dam/tag/[assetId]/route.ts:19` skips the
   secret check when `DAM_TAG_SECRET` is unset. Carry-forward from prior audit.
   _(Warden MEDIUM)_

### P2 — backlog / hygiene

- `operator/intelligence/drilldown` missing `requireRole` — READ_ONLY members can pull
  sponsor analytics. _(Warden MEDIUM)_
- Single `CHECKIN_SECRET` spans all workspaces; tenant comes from a user-supplied
  `?workspace=` param. _(Spine #5)_
- Public apply generic endpoint resolves tenant via `findFirst()` (no filter) — breaks
  the moment a 2nd tenant exists. `apply/membership/route.ts:46`. _(Spine #4)_
- Agent system prompt hardcodes "No Bad Company" — multi-tenant expansion blocker.
  `lib/agent/system-prompt.ts:7`. _(Spine #10)_
- `GET /api/operator/members` has no pagination. _(Spine #7)_
- `RSVP` dual status fields (`status` enum + raw `ticketStatus` string, unindexed).
  `QAMission` uses raw strings for enum-worthy fields. _(Flux #6, #5)_
- Unbounded `customAnswers` in new member-portal access/payment routes. _(Warden LOW)_
- 2 unmerged branches (`claude/sync-context-2026-05-28`, `feat/dam-video-play-overlay`)
  carry real work at risk of loss. _(Proof #5)_
- `nobc-house-phone/` and `_context/SECURITY_AUDIT.md` untracked — no history, creds
  risk one `git add .` away. Commit or gitignore deliberately. _(Proof #6, #7)_

## Strengths (don't regress these)

- Tenant isolation is real and consistent: `requireRole()` + `requireWorkspaceId()` at
  the top of authed routes; 45/49 models carry `workspaceId NOT NULL @@index`.
- Thin route handlers over a real `lib/` service layer; Zod validation on public surfaces.
- Stripe + Clerk webhooks verify signatures; timing-safe comparisons present.
- Conservative `onDelete` (mostly `Restrict`), narrow intentional cascades.
- Low type-escape rate (1 `@ts-expect-error`, ~6 `as any` across 544 files); `tsc` clean.
- A prior security audit already exists and closed real findings (e.g. the approve-route IDOR).

## Recommended starting point

Two parallel tracks. **Safety track:** P0 #1 (apply migration) + #2/#3 (check-in secret,
plus-one bypass) — small, high-leverage, shippable in a day. **Durability track:** stand up
the CI gate (P0 #4) so nothing regresses while the rest is fixed. Everything else sequences
behind those.
